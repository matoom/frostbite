Frostbite

A frontend for Dragonrealms MUD

4 Development

The Frostbite frontend application has been programmed in C/C++ using Qt framework version 4.7 (updated to compile with 5.2). The development project is set up in Qt Creator IDE. All the Qt tools are free to use for open source projects and can be downloaded from the Qt website.


Github

The application source code is hosted in the Github repository - https://github.com/matoom/frostbite. It is possible to contribute to the Frostbite project in Github by forking the repository as it is described in the following guide - https://help.github.com/articles/fork-a-repo


Setting up the project

The following steps are necessary to set up the project in Qt Creator IDE in Windows:

  • Open Frostbite.pro in Qt Creator;
  • Load the environment settings -> No -> Select Desktop project;
  • In environment.h -> change project path for offline mock file - #define MOCK_DATA_PATH "C:/Projects/FrostBite/support/mock.xml";
  • in main.cpp -> set MainWindow::DEBUG = true;
  • Build for debug target;
  • In projects tab -> add a custom executable -> find exe file in debug build folder;
  • Run project in debug mode.

High level architecture

The entry point for the Frostbite application is main.cpp which runs the mainwindow.cpp application.

The MainWindow class holds instances for some of the main purpose models like:

  • windowmanager.cpp - everything related to game windows
  • toolbarmanager.cpp - handles items on the toolbar
  • timerbar.cpp - holds timer bar logic
  • commandline.cpp - the client command line
  • connectionmanager.cpp - everything related to game connection and data handling
  • scriptservice.cpp - handles scripting processes
  • menuhandler.cpp - handles client main menu logic
